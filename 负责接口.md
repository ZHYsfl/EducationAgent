所有人按 `系统接口规范.md` 实现，**字段名和路径不能自创**。

- 返回格式统一 `code/message/data`。
- ID 统一前缀（`user_`/`task_`/`page_`/`doc_`...），时间戳统一毫秒。
- Voice Agent 不阻塞外部服务：KB、Memory、Web Search 都是异步回注。

---

## 1) 周浩洋（Voice Agent + 编排层）负责的接口

- `WS /ws`  
实时语音主通道，前端发 `vad_start/vad_end/音频`，你要回 `status/transcript/response/音频`，并扩展需求收集消息（`requirements_progress`、`requirements_summary`、`requirements_confirm`）。
- `POST /api/v1/upload`（网关）  
前端只调这个上传入口；你这边做网关转发到数据库服务的 `POST /api/v1/files/upload`，并把结果回给前端。
- `GET /api/v1/tasks/{task_id}/preview`  
前端预览统一从你这里拿，你负责聚合 PPT 状态并返回可展示的 `page_order/pages_info`。
- `POST /api/v1/voice/ppt_message`（兼容别名 `POST /api/v1/voice/ppt_message_tool`）  
PPT Agent 反向"打断求助"入口；你收到后按优先级入队，`conflict_question` 要立刻走语音播报。
- （调用职责）调用 `POST /api/v1/ppt/init`  
先通过多轮追问把需求收满并确认，再发起初始化，不能用户一句"做个PPT"就直接开工。
- （调用职责）调用 `POST /api/v1/ppt/feedback`  
把用户自然语言降维成 `intents` 发给 PPT Agent，冲突回复要带 `reply_to_context_id`。
- （调用职责）异步调用 `POST /api/v1/kb/query`、`POST /api/v1/memory/recall`、`POST /api/v1/search/query`  
必须 goroutine 异步，不许卡住语音主线程。goroutine 拿到结果后塞进 `contextQueue`。
- （调用职责）异步调用 `GET /api/v1/memory/profile/{user_id}`  
需求收集开始时预填用户已知偏好，减少重复追问。
- （调用职责）`POST /api/v1/canvas/vad-event`  
用户 `vad_start` 且当前已有活跃 `task_id` 时，异步通知 PPT Agent（Body：`task_id`、`timestamp`、`viewing_page_id`），供画布快照锚定；无任务时不发。
- （调用职责）轮询 `GET /api/v1/search/results/{request_id}`  
Web 搜索返回 `pending` 时，goroutine 里轮询直到拿到结果。
- （调用职责）搜索结果沉淀到知识库  
KB 分数低且 Web Search 有结果时，goroutine 异步调用 `POST /api/v1/kb/ingest-from-search` 把搜索精华写回用户知识库。
- 多 PPT 任务路由（见规范 §0.5）  
Session 维护 `active_task_id`，优先级：冲突回复 > 明确提及 > 当前活跃 > 唯一任务 > 追问，绝不猜。

## 2) 钟天贻（PPT Agent + 任务管理）负责的接口

**PPT Agent：**

- `POST /api/v1/ppt/init`  
接收已确认的结构化需求（含 `teaching_elements` + `description`），创建 `task_id`，启动首稿生成。
- `POST /api/v1/ppt/feedback`  
接收 Voice Agent 的 `intents`，做页面修改/插入/删除/全局改；冲突进入三路合并与悬挂流程。
- `GET /api/v1/canvas/status?task_id={task_id}`  
返回当前任务的页面路由表和每页状态，给前端轮询预览和 Voice Agent "看当前页"用。
- `POST /api/v1/canvas/vad-event`  
接收 Voice Agent 的 VAD 触发信号（`task_id`、`timestamp`、`viewing_page_id`），据此做画布快照/版本锚定（与规范 §3.1 `VADEvent` 一致）；可内部写 Redis `snapshot:{task_id}:{timestamp}`。
- `POST /api/v1/ppt/export`  
把任务导出成 `pptx/docx/html5`，返回 `export_id`，异步执行。
- `GET /api/v1/ppt/export/{export_id}`  
导出状态查询，完成后给出 `download_url`。
- `GET /api/v1/ppt/page/{page_id}/render?task_id={task_id}`  
单页渲染结果与版本，便于前端做页级刷新和调试定位。
- （调用职责）`POST /api/v1/voice/ppt_message`  
发生冲突或关键状态变化时反向通知 Voice Agent，冲突问题必须带 `context_id`。

**Tasks（任务元数据，与 PPT 任务状态联动）：**

- `POST /api/v1/tasks`  
创建任务元数据，和 PPT Agent 的 `task_id` 保持一致。
- `GET /api/v1/tasks/{task_id}`  
查询任务详情与状态。
- `PUT /api/v1/tasks/{task_id}/status`  
任务状态机更新入口（`pending/generating/completed/failed/exporting`）。
- `GET /api/v1/tasks?session_id={session_id}&page=1&page_size=20`  
会话下任务列表分页。

## 3) 段怡鹏（Knowledge Base）负责的接口

- `POST /api/v1/kb/collections`  
创建知识库集合（按学科/用户管理）。
- `GET /api/v1/kb/collections?user_id={user_id}`  
列出用户已有集合，供前端或编排层选择检索范围。
- `POST /api/v1/kb/documents`  
把已上传文件做"异步索引入库"任务（解析、切块、向量化）。
- `GET /api/v1/kb/documents/{doc_id}`  
返回索引进度（`processing/indexed/failed`），让上层知道什么时候可检索。
- `POST /api/v1/kb/query`  
核心 RAG 接口，按 query 返回相关 chunks 和分数，给 Voice Agent 注入上下文。
- `DELETE /api/v1/kb/documents/{doc_id}`  
删除文档时要把向量库对应 chunks 一并清掉。
- `GET /api/v1/kb/collections/{collection_id}/documents?page=1&page_size=20`  
集合内文档管理页需要这个接口做列表和分页。
- `POST /api/v1/kb/parse`  
纯解析接口（不入向量库），给 PPT 参考资料解析等场景复用。
- `POST /api/v1/kb/ingest-from-search`  
接收 Web Search 沉淀的搜索结果，按 URL 去重后切块入向量库（`doc_type: "web_snippet"`），实现"搜→存→下次 RAG 直接命中"的闭环。

## 4) 王松彬（用户与记忆服务）负责的接口

**Auth（用户认证）：**

- `POST /api/v1/auth/register`  
  用户注册，落库并返回登录 token。

- `POST /api/v1/auth/login`  
  账号认证，签发 token 给前端和上层服务使用。

- `POST /api/v1/auth/verify` 

  

- `GET /api/v1/auth/profile`  
返回当前用户基础资料，供前端和编排层显示。

**Memory（记忆）：**

- `POST /api/v1/memory/extract`  
从对话提取"事实+偏好+摘要"，写入长期/短期记忆。
- `POST /api/v1/memory/recall`  
按当前 query 召回相关记忆给 Voice Agent 用于个性化回答与追问。
- `GET /api/v1/memory/profile/{user_id}`  
读取用户画像，给需求收集阶段做预填（少问重复问题）。
- `PUT /api/v1/memory/profile/{user_id}`  
更新画像字段，支持偏好纠偏。
- `POST /api/v1/memory/working/save`  
保存当前会话工作记忆（Redis），支撑多轮上下文稳定。
- `GET /api/v1/memory/working/{session_id}`  
取回会话工作记忆，支撑中断恢复和继续追问。

## 5) 曾晨曦（存储、会话与搜索服务）负责的接口

**Files（文件管理）：**

- `POST /api/v1/files/upload`  
真正的文件上传落地接口，写对象存储并保存 `files` 元数据。
- `GET /api/v1/files/{file_id}`  
返回文件元信息和下载地址（必要时签名 URL）。
- `DELETE /api/v1/files/{file_id}`  
删除文件记录并触发对象存储清理任务。
- （额外要求）文件清理触发器 / Outbox  
`files` 记录被删时必须写 `file_delete_jobs`，后台 worker 删除对象存储，避免垃圾文件。

**Sessions（会话管理）：**

- `POST /api/v1/sessions`  
创建会话（一段教学对话的容器）。
- `GET /api/v1/sessions/{session_id}`  
读取单个会话详情。
- `GET /api/v1/sessions?user_id={user_id}&page=1&page_size=20`  
会话列表分页查询。
- `PUT /api/v1/sessions/{session_id}`  
更新会话标题/状态（归档、完成等）。

**Web Search：**

- `POST /api/v1/search/query`  
发起异步搜索，返回 `request_id` 或直接结果；结果需要做摘要精炼，确保 `snippet` 质量可入知识库（非原始 HTML）。
- `GET /api/v1/search/results/{request_id}`  
轮询异步搜索结果，统一返回 `pending/completed/failed`。
- （可选增强）内部调用 `POST /api/v1/kb/query` 去重  
避免把知识库已有内容重复回注给 Voice Agent，减少噪声。

---

每个人先把自己负责接口做成可调通的最小闭环（happy path），然后我们做跨服务联调；联调时只认文档字段，不认口头约定。
