| 组件 | 版本/配置 |
|------|----------|

| CUDA Driver | 13.0 |

## 一、WSL2 环境搭建

### 1. 安装 WSL2

```powershell
# PowerShell 管理员模式
wsl --install -d Ubuntu
```

重启后，设置 Ubuntu 用户名密码。

### 2. 确认 CUDA 可用

```bash
nvidia-smi
```

应看到 RTX 5080 和 CUDA 13.0 的信息。

## 二、安装 vLLM

### 1. 安装 uv（Python 包管理器）

```bash
curl -LsSf https://astral.sh/uv/install.sh | sh
source $HOME/.local/bin/env
```

### 2. 创建虚拟环境

```bash
# 创建带基础工具的虚拟环境
uv venv --python 3.12 --seed
source .venv/bin/activate
```

### 3. 安装 vLLM

```bash
# 安装 CUDA 12.8 版本（支持 Blackwell/5080）
uv pip install vllm --torch-backend=cu128
```

> 当前 PyTorch 没有 cu130 (CUDA 13.0) 版本，cu128 是 Blackwell 支持的最低版本。

## 四、启动 vLLM 服务

### 1. 基础对话模式

```bash
vllm serve /mnt/d/创业/myResearch/EducationAgent/models/Qwen3___5-0___8B \
  --port 8000 \
  --tensor-parallel-size 1 \
  --max-model-len 32768 \
  --dtype auto \
  --host 0.0.0.0
```

### 2. 工具调用模式（支持 Function Calling）

```bash
vllm serve /mnt/d/创业/myResearch/EducationAgent/Qwen3___5-0___8B \
  --port 8000 \
  --tensor-parallel-size 1 \
  --max-model-len 32768 \
  --dtype auto \
  --enable-auto-tool-choice \
  --tool-call-parser qwen3_coder \
  --host 0.0.0.0
```

**关键参数说明**：

| 参数 | 说明 |
|------|------|
| `--host 0.0.0.0` | 允许外部访问（默认只允许本地） |
| `--enable-auto-tool-choice` | 启用工具自动选择 |
| `--tool-call-parser qwen3_coder` | Qwen3 专用工具解析器 |
| `--max-model-len 32768` | 最大上下文长度 |
| `--dtype auto` | 自动选择数据类型（BF16/FP16） |

## 六、公网暴露方案

### 方案 1：Cloudflare Tunnel（推荐，免费、安全）

无需公网 IP，无需配置防火墙，HTTPS 自动证书。

```bash
# 1. 安装 cloudflared
wget -q https://github.com/cloudflare/cloudflared/releases/latest/download/cloudflared-linux-amd64 -O cloudflared
chmod +x cloudflared
sudo mv cloudflared /usr/local/bin/

# 2. 登录 Cloudflare 账号
cloudflared tunnel login

# 3. 创建隧道
cloudflared tunnel create vllm-server

# 4. 配置隧道（编辑 ~/.cloudflared/config.yml）
echo "tunnel: <隧道ID>

credentials-file: ~/.cloudflared/<隧道ID>.json

ingress:
  - hostname: vllm.yourdomain.com
    service: http://localhost:8000
  - service: http_status:404" > ~/.cloudflared/config.yml

# 5. 启动隧道
cloudflared tunnel run vllm-server
```

然后就可以通过 `https://vllm.yourdomain.com/v1` 访问你的 API。

### 方案 2：ngrok（临时测试）

```bash
# 安装 ngrok
wget https://bin.equinox.io/c/bNyj1mQVY4c/ngrok-v3-stable-linux-amd64.tgz
tar xvzf ngrok-v3-stable-linux-amd64.tgz
sudo mv ngrok /usr/local/bin/

# 注册并配置 authtoken
ngrok config add-authtoken <你的token>

# 暴露 8000 端口
ngrok http 8000
```

会给你一个临时公网 URL（如 `https://abc123.ngrok.io`），可以直接访问。

### 方案 3：frp（有公网服务器时）

如果你有云服务器，可以用 frp 做内网穿透：

**frps（公网服务器）配置**：
```ini
[common]
bind_port = 7000
token = your_token
```

**frpc（本地 WSL2）配置**：
```ini
[common]
server_addr = x.x.x.x  # 你的公网服务器IP
server_port = 7000
token = your_token

[vllm]
type = tcp
local_ip = 127.0.0.1
local_port = 8000
remote_port = 8000
```

### 方案 4：公网 IP + 防火墙（不推荐，不安全）

如果你有公网 IP：

1. 路由器配置端口转发（8000 → 你的电脑IP:8000）
2. Windows 防火墙放行 8000 端口
3. **必须添加 API Key 验证**，否则任何人都能访问

```bash
# 启动时添加 API Key（vLLM 0.17+ 支持）
vllm serve ... --api-key "your-secret-key"
```

## 七、生产环境建议

### 性能优化

```bash
# 启用 Multi-Token Prediction (MTP) 加速
vllm serve ... \
  --speculative-config '{"method":"qwen3_next_mtp","num_speculative_tokens":2}'

# 启用连续批处理
vllm serve ... --enable-chunked-prefill
```

### 安全加固

1. **必须启用 API Key**：`--api-key "your-secret-key"`
2. **使用 HTTPS**：通过 Cloudflare/ngrok 自动获得
3. **限制访问 IP**：配合 Nginx 或防火墙
4. **监控资源**：使用 `nvidia-smi dmon` 监控 GPU 使用率

### 开机自启（systemd）

创建 `/etc/systemd/system/vllm.service`：

```ini
[Unit]
Description=vLLM Service
After=network.target

[Service]
Type=simple
User=zane
WorkingDirectory=/home/zane
Environment="PATH=/home/zane/.venv/bin"
ExecStart=/home/zane/.venv/bin/vllm serve /mnt/d/创业/myResearch/EducationAgent/Qwen3___5-0___8B --port 8000 --host 0.0.0.0 --enable-auto-tool-choice --tool-call-parser qwen3_coder
Restart=always

[Install]
WantedBy=multi-user.target
```

然后：
```bash
sudo systemctl daemon-reload
sudo systemctl enable vllm
sudo systemctl start vllm
```

## 八、常见问题

### 1. 第一次请求慢，后续快

这是正常的，第一次需要加载模型到 GPU 显存。后续请求会保持 warmed up 状态。

### 2. Windows 直接运行报错

vLLM 不支持原生 Windows，必须使用 WSL2。

### 3. CUDA Out of Memory

减小 `--max-model-len` 或使用量化版本：
```bash
--quantization awq  # 或 gptq
```

### 4. 模型 ID 404 错误

使用本地路径作为 model ID：
```python
model="/mnt/d/创业/myResearch/EducationAgent/Qwen3___5-0___8B"
```

## 九、Voice Agent 四模型完整部署

### VRAM 分配（RTX 5080 16GB）

| 服务 | 模型 | 估计 VRAM | 端口 |
|------|------|----------|------|
| vLLM (Small + Large LLM 共用) | Qwen3.5-0.8B | ~1.6 GB | 8000 |
| Qwen3-ASR WebSocket | Qwen3-ASR-0.6B | ~1.2 GB | 10096 |
| CosyVoice3 FastAPI | Fun-CosyVoice3-0.5B | ~1.2 GB | 50000 |
| **合计** | | **~4.0 GB** | |

> Qwen3-ASR-0.6B 比 Fun-ASR-Nano (0.8B) 更省显存，且效果更好。

> Voice Agent Go binary 在 Windows 侧运行，通过 `localhost` 访问 WSL 内三个服务。

---

### A. vLLM — Small LLM + Large LLM 共用（port 8000）

Small LLM 和 Large LLM 均为 Qwen3.5-0.8B，共用同一个 vLLM 实例。
**不加 `--reasoning-parser`**：让 `<think>` 标签保持在 `Content` 字段内，由 Go 侧 `thinkFilter` 处理；若用 reasoning-parser，思考内容会被分离到 `reasoning` 字段，`StreamChat` 读不到，打断保存历史功能失效。

```bash
source .venv/bin/activate

vllm serve /mnt/d/创业/myResearch/EducationAgent/models/Qwen3___5-0___8B \
  --port 8000 \
  --host 0.0.0.0 \
  --gpu-memory-utilization 0.35 \
  --max-model-len 8192 \
  --dtype auto
```

验证：
```bash
curl http://localhost:8000/v1/models
```

---

### B. Qwen3-ASR — ASR WebSocket 服务（port 10096）

Qwen3-ASR 是阿里云开源的 ASR 模型，支持 vLLM 后端，比 Fun-ASR-Nano 更轻量、效果更好。

#### 1. 安装依赖

```bash
source .venv/bin/activate

# 安装 qwen-asr（vLLM 后端）
uv pip install -U qwen-asr[vllm]

# 可选：安装 FlashAttention 加速
uv pip install flash-attn --no-build-isolation
```

#### 2. 下载模型（国内用 ModelScope）

```bash
# 使用 ModelScope 下载（国内更快）
modelscope download --model Qwen/Qwen3-ASR-0.6B --local_dir ./models/asr

# 或下载 1.7B 版本（效果更好）
# modelscope download --model Qwen/Qwen3-ASR-1.7B --local_dir ./models/asr
```

#### 3. 启动服务

```bash
python3 /mnt/d/创业/myResearch/EducationAgent/deploy/start_asr_vllm.py \
  --model /mnt/d/创业/myResearch/EducationAgent/models/asr \
  --port 10096 \
  --gpu-util 0.25
```

**或使用 HuggingFace ID 自动下载：**
```bash
python3 /mnt/d/创业/myResearch/EducationAgent/deploy/start_asr_vllm.py \
  --model Qwen/Qwen3-ASR-0.6B \
  --port 10096
```

### D. Voice Agent .env 配置

`voice_agent/.env`（复制 `.env.example` 后修改）：

ASR 和 TTS 均支持 **local**（本地自建）和 **remote**（豆包云端 API）两种模式，通过 `ASR_MODE` 和 `TTS_MODE` 切换。

#### 模式一：全本地部署（需要 GPU）

```env
# LLM — Small 和 Large 共用 port 8000，同一个 vLLM 实例
SMALL_LLM_BASE_URL=http://localhost:8000/v1
SMALL_LLM_MODEL=Qwen3___5-0___8B
LARGE_LLM_BASE_URL=http://localhost:8000/v1
LARGE_LLM_MODEL=Qwen3___5-0___8B

# ASR — 本地 WebSocket
ASR_MODE=local
ASR_WS_URL=ws://localhost:10096

# TTS — 本地 CosyVoice2
TTS_MODE=local
TTS_URL=http://localhost:50000

# Server
SERVER_PORT=9000
```

#### 模式二：ASR + TTS 走豆包云端 API（不占 GPU）

无需部署 ASR/TTS 服务，只需 vLLM 一个 GPU 服务。
需要先在 [火山引擎控制台](https://console.volcengine.com/) 开通豆包语音服务，获取 AppKey/Token 等凭证。

```env
# LLM — 同上
SMALL_LLM_BASE_URL=http://localhost:8000/v1
SMALL_LLM_MODEL=Qwen3___5-0___8B
LARGE_LLM_BASE_URL=http://localhost:8000/v1
LARGE_LLM_MODEL=Qwen3___5-0___8B

# ASR — 豆包大模型流式语音识别
ASR_MODE=remote
DOUBAO_ASR_APP_KEY=你的AppKey
DOUBAO_ASR_ACCESS_KEY=你的AccessKey
DOUBAO_ASR_RESOURCE_ID=volc.bigasr.sauc.duration

# TTS — 豆包在线语音合成
TTS_MODE=remote
DOUBAO_TTS_APPID=你的AppId
DOUBAO_TTS_TOKEN=你的Token
DOUBAO_TTS_CLUSTER=volcano_tts
DOUBAO_TTS_VOICE_TYPE=BV700_streaming

# Server
SERVER_PORT=9000
```

> 也可以混合模式，如 `ASR_MODE=remote` + `TTS_MODE=local`（ASR 走云端，TTS 用本地 CosyVoice2）。

### 常见问题

**Q: `nvidia-smi` 在 WSL 可见但 PyTorch 找不到 GPU**

```bash
# 确认 CUDA 在 WSL 环境变量中
export CUDA_VISIBLE_DEVICES=0
python3 -c "import torch; print(torch.cuda.is_available())"
```

**Q: vLLM 显存不够**

减小 `--gpu-memory-utilization`（如 0.25）或减小 `--max-model-len`（如 4096）。

**Q: ASR 连接超时**

检查 `start_asr_vllm.py` 是否成功加载模型（首次加载需 10-20 秒）。
如果报错 `No module named 'qwen_asr'`，需要安装：`uv pip install -U qwen-asr[vllm]`

**Q: TTS 无声音**

确认 `--ref-wav` 路径存在且为 16kHz/24kHz WAV 文件。


export PYTORCH_CUDA_ALLOC_CONF=expandable_segments:True

## vLLM Docker 启动（Windows 推荐）

下次运行 vLLM，在 PowerShell 中执行：

```powershell
docker run --gpus all -p 8000:8000 `
  -v D:\创业\myResearch\EducationAgent\models:/models `

  --tensor-parallel-size 1 `
  --dtype auto `
  --max-model-len 8192 `
  --gpu-memory-utilization 0.82 `

```

或单行（复制整行）：

```powershell
docker run --gpus all -p 8000:8000 -v D:\创业\myResearch\EducationAgent\models:/models vllm/vllm-openai:latest --model /models/Qwen3___5-0___8B --tensor-parallel-size 1 --dtype auto --max-model-len 8192 --gpu-memory-utilization 0.82 --enforce-eager
```

启动后，`.env` 中 `SMALL_LLM_MODEL` 和 `LARGE_LLM_MODEL` 需设为 `/models/Qwen3___5-0___8B`。
