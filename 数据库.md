# 一、表规范

这份规范里，**已经明确给出 DDL 的核心表**有：

1. `users`
2. `pending_registrations`
3. `sessions`
4. `tasks`
5. `files`
6. `kb_documents`
7. `memory_entries`
8. `memory_cards`
9. `memory_dialogue_chunks`
10. `memory_objects`

因为文档其他章节还显式定义了这些持久化对象，只是没有给出最终 DDL：

- PPT 页级数据（`page_id`, `render_url`, `status`, `version`, `py_code`）
- 任务需求采集数据（`TaskRequirements`）
- 参考文件与任务的结构化关联（`ReferenceFileReq` / `ReferenceFile`）
- KB Collection（`KBCollection`）
- 文档切块元数据（`TextChunk` / `ChunkMeta`）
- 导出任务（`export_id`, `format`, `download_url`, `file_size`）
- 页面挂起/冲突处理状态（`SuspendedPage`）
- 页面合并队列状态（`PageMergeState`）
- 异步删除作业（文档里要求 `file_delete_jobs` / `memory_object_delete_jobs`）

# 二、规范已明确的正式数据库字段定义

## 1) 用户表 `users`

用途：系统登录用户、教师/管理员主体。

| 字段名          | 类型           | 必填 | 说明                                       |
| --------------- | -------------- | ---- | ------------------------------------------ |
| `id`            | `VARCHAR(64)`  | 是   | 主键，格式 `user_<uuid>`                   |
| `username`      | `VARCHAR(64)`  | 是   | 用户名，唯一                               |
| `email`         | `VARCHAR(128)` | 是   | 邮箱，唯一                                 |
| `password_hash` | `VARCHAR(256)` | 是   | 密码哈希                                   |
| `display_name`  | `VARCHAR(128)` | 否   | 显示名称，默认空                           |
| `subject`       | `VARCHAR(64)`  | 否   | 教学学科                                   |
| `school`        | `VARCHAR(128)` | 否   | 学校名称                                   |
| `role`          | `VARCHAR(16)`  | 是   | 角色，默认 `teacher`，可选 `teacher/admin` |
| `created_at`    | `BIGINT`       | 是   | 创建时间，Unix ms                          |
| `updated_at`    | `BIGINT`       | 是   | 更新时间，Unix ms                          |

约束：

- PK: `id`
- UNIQUE: `username`
- UNIQUE: `email`

# 三、从接口规范强制推导出的表

## 11) PPT 页面表 `task_pages`

原因：

- 规范里大量出现 `page_id`、`task_id`、`render_url`、`status`、`version`、`py_code`、`updated_at`
- `preview`、`canvas/status`、`page render query` 都依赖页级持久化数据。

### 建议字段定义

| 字段名        | 类型            | 必填 | 说明                                             |
| ------------- | --------------- | ---- | ------------------------------------------------ |
| `id`          | `VARCHAR(64)`   | 是   | 主键，`page_<uuid>`                              |
| `task_id`     | `VARCHAR(64)`   | 是   | 所属任务                                         |
| `page_index`  | `INT`           | 是   | 当前页序号                                       |
| `status`      | `VARCHAR(32)`   | 是   | `rendering/completed/failed/suspended_for_human` |
| `render_url`  | `VARCHAR(1024)` | 否   | 渲染图 URL                                       |
| `py_code`     | `TEXT`          | 否   | 当前页面 Python 源代码                           |
| `version`     | `INT`           | 是   | 渲染版本号，默认 1                               |
| `last_update` | `BIGINT`        | 是   | 给前端预览接口用                                 |
| `updated_at`  | `BIGINT`        | 是   | 更新时间                                         |
| `created_at`  | `BIGINT`        | 是   | 创建时间                                         |

建议约束：

- FK: `task_id -> tasks(id)` `ON DELETE CASCADE`
- UNIQUE: `(task_id, page_index)`

建议索引：

- `(task_id)`
- `(task_id, status)`
- `(task_id, page_index)`

| `page_id`    | `VARCHAR(64)` | 是   | 页面 ID  |

- UNIQUE: `(task_id, sort_order)`

## 13) 任务需求表 `task_requirements`

原因：

- `TaskRequirements` 是完整的结构化需求对象
- 这块如果只存在 Session 内存里，后续确认、回退、审计、断点恢复会很痛。

### 建议字段定义

| 字段名               | 类型           | 必填 | 说明                                         |
| -------------------- | -------------- | ---- | -------------------------------------------- |
| `id`                 | `VARCHAR(64)`  | 是   | 可用 `req_<uuid>`，或直接一任务一条          |
| `session_id`         | `VARCHAR(64)`  | 是   | 会话 ID                                      |
| `user_id`            | `VARCHAR(64)`  | 是   | 用户 ID                                      |
| `task_id`            | `VARCHAR(64)`  | 否   | 确认生成后回填                               |
| `topic`              | `VARCHAR(256)` | 否   | 课程主题                                     |
| `knowledge_points`   | `JSONB`        | 否   | 核心知识点数组                               |
| `teaching_goals`     | `JSONB`        | 否   | 教学目标数组                                 |
| `teaching_logic`     | `TEXT`         | 否   | 教学逻辑                                     |
| `target_audience`    | `VARCHAR(256)` | 否   | 目标受众                                     |
| `key_difficulties`   | `JSONB`        | 否   | 重难点数组                                   |
| `duration`           | `VARCHAR(64)`  | 否   | 时长                                         |
| `total_pages`        | `INT`          | 否   | 页数                                         |
| `global_style`       | `VARCHAR(128)` | 否   | 全局风格                                     |
| `interaction_design` | `TEXT`         | 否   | 互动设计                                     |
| `output_formats`     | `JSONB`        | 否   | 输出格式数组                                 |
| `additional_notes`   | `TEXT`         | 否   | 补充说明                                     |
| `collected_fields`   | `JSONB`        | 否   | 已采集字段列表                               |
| `status`             | `VARCHAR(32)`  | 是   | `collecting/confirming/confirmed/generating` |
| `created_at`         | `BIGINT`       | 是   | 创建时间                                     |
| `updated_at`         | `BIGINT`       | 是   | 更新时间                                     |

建议：

- `knowledge_points`、`teaching_goals`、`key_difficulties`、`output_formats`、`collected_fields` 用 `JSONB`
- 若想保持范式，可再拆子表；但当前阶段 `JSONB` 最适合多 agent 协作
