# PPT Agent 服务 — 联动生成实现文档

> 本文档记录 2026-04-07 本次迭代中新增/修改的所有代码实现逻辑，适用于曾晨曦负责的 PPT Agent 服务模块。

---

## 一、整体架构：PPT Init 联动生成

### 1.1 设计目标

根据【开发说明】4a/4b/4c 的要求，PPT 初稿生成时应同时产出配套教案和动画/游戏。设计采用**分离并发**模式：

- PPT 生成是主流程，完成后才返回 `task_id` 给 Voice Agent
- 教案和动画/游戏在 PPT 生成 goroutine 中**独立启动新 goroutine 并发执行**，互不阻塞
- 三者均失败互不影响，Teacher 可分别轮询各自状态

### 1.2 调用时序图

```
Voice Agent                              PPT Agent 服务
     |                                        |
     |--- POST /api/v1/ppt/init ------------->|
     |    (含 auto_generate_teaching_plan     |
     |     auto_generate_content_diversity)   |
     |                                        |
     |    202 Accepted {task_id}             |
     |<---------------------------------------|
     |                                        |
     |                           ┌─ PPT 主 goroutine ────────────────┐
     |                           │ 1. 创建 Task / Canvas            │
     |                           │ 2. LLM 生成页面 py_code            │
     |                           │ 3. 并发 Render 各页面              │
     |                           │ 4. UpdateStatus(complete, 100)    │
     |                           │ 5. 启动教案子 goroutine ─────────┐ │
     |                           │ 6. 启动动画子 goroutine ─────────┐│ │
     |                           └──────────────────────────────────┘│ │
     |                                                                   │ │
     |── GET /api/v1/teaching_plan/status/:id ─────────────────────────│─|
     |<──────────────────────────────────────────────────────────────│──|
     |                                                                   |
     |--- GET /api/v1/content_diversity/status/:id ───────────────────│-|
     |<──────────────────────────────────────────────────────────────│--|
```

### 1.3 核心请求/响应字段

**PPTInitRequest** 新增字段：

```go
// PPTAgent 自动联动开关
AutoGenerateTeachingPlan     bool   // true → Init 完成后自动触发教案生成
AutoGenerateContentDiversity  bool   // true → Init 完成后自动触发动画/游戏生成
ContentDiversityType          string // "animation" | "game" | "both"，默认 "both"
ContentDiversityGameType       string // 小游戏类型，默认 "random"
ContentDiversityAnimationStyle string // 动画风格，默认 "all"
```

**Task 模型新增字段**：

```go
PlanID          string // 关联的教案任务 ID
ContentResultID string // 关联的内容多样性任务 ID
```

## 二、模型层变更

### 2.1 `internal/model/ppt.go` — PPTInitRequest

新增 5 个字段，实现 Init 请求中对教案/动画/游戏的联动控制。

```go
type PPTInitRequest struct {
    // ... 原有字段 ...
    AutoGenerateTeachingPlan     bool   // 联动开关：教案
    AutoGenerateContentDiversity bool   // 联动开关：动画/游戏
    ContentDiversityType          string // 内容类型
    ContentDiversityGameType     string // 游戏类型
```

### 2.2 `internal/model/task.go` — Task

用于记录 PPT Task 与教案/动画的关联关系，方便前端通过 `task_id` 查找所有产出。

```go
type Task struct {
    // ... 原有字段 ...
    PlanID           string // 关联教案任务 ID
    ContentResultID  string // 关联内容多样性任务 ID
}
```

### 2.3 `internal/repository/task_repository.go` — 接口扩展

`TaskRepository` 接口新增两个方法，`InMemoryTaskRepository` 和 `RedisTaskRepository` 均实现：

```go
UpdatePlanID(taskID, planID string) error
UpdateContentResultID(taskID, resultID string) error
```

子任务 ID 在其生成完成后由 goroutine 写回主 Task 记录。

## 三、服务层变更

### 3.1 `internal/service/ppt_service.go`

#### 新增依赖注入方法

```go
func (s *PPTService) AttachTeachingPlanService(ts *TeachingPlanService)
func (s *PPTService) AttachContentDiversityService(cds *ContentDiversityService)
```

在 `cmd/server/main.go` 中初始化时调用，将两个服务注入 PPTService：

```go
pptService.AttachTeachingPlanService(teachingPlanService)
pptService.AttachContentDiversityService(contentDiversityService)
```

#### `runInitGeneration` 中的联动逻辑

PPT 主 goroutine 在 `UpdateStatus(complete, 100)` 之后，**非阻塞地**启动两个子 goroutine：

```go
// PPT 主 goroutine 执行完毕后：
if req.AutoGenerateTeachingPlan && s.teachingPlanService != nil {
    go func() {
        planResp, _ := s.teachingPlanService.Generate(ctx, planReq)
        _ = s.taskRepo.UpdatePlanID(taskID, planResp.PlanID) // 写回
    }()
}
if req.AutoGenerateContentDiversity && s.contentDiversityService != nil {
    go func() {
        divResp, _ := s.contentDiversityService.Generate(ctx, divReq)
        _ = s.taskRepo.UpdateContentResultID(taskID, divResp.ResultID) // 写回
    }()
}
```

关键设计点：
- 两个 goroutine 并发执行，不阻塞 PPT 主流程
- 即使子生成失败，PPT 仍标记为 `completed`
- 通过 `UpdatePlanID` / `UpdateContentResultID` 将子任务 ID 同步回 Task

### 3.2 `internal/service/content_diversity_service.go` — GIF/MP4 导出

原有内嵌 Python 脚本（通过 `fmt.Sprintf` 动态生成）存在以下问题：
- `page.video.save_as()` 在 playwright 新版中 API 已变更
- 无法复用 `render_animation.py` 的完善逻辑

**修改方案**：将内嵌脚本替换为对独立 `render_animation.py` 的进程调用，通过 stdin 传入 JSON：

```go
inputJSON := map[string]any{
    "output_path": outputPath,
    "format":      format, // "gif" | "mp4"
    "title":       contentID,
    "data": map[string]any{
        "html_content": htmlContent,
    },
}
inputBytes, _ := json.Marshal(inputJSON)

cmd := exec.CommandContext(cmdCtx, pythonPath, animScriptPath)
cmd.Stdin = bytes.NewReader(inputBytes) // JSON 通过 stdin 传入
_ = cmd.Run()
```

`render_animation.py` 内部根据 `format` 参数决定走 GIF 路径（playwright 截图帧合成）或 MP4 路径（playwright 视频录制）。

## 四、渲染脚本（新增）

三个独立 Python 脚本均位于 `zcxppt/internal/infra/renderer/` 目录，通过 stdin/stdout 与 Go 服务交互。

### 4.1 `render_teaching_plan.py` — Word 教案渲染

#### 输入格式（stdin JSON）

```json
{
  "output_path": "/path/to/output.docx",
  "plan": "{\"title\":\"...\",\"teaching_goals\":[...],\"teaching_process\":{...}}"
}
```

`plan` 字段为 LLM 生成的教案结构化 JSON 字符串，主要结构：

```json
{
  "title": "教案标题",
  "subject": "数学",
  "grade": "高一",
  "duration": "45分钟",
  "teaching_goals": ["掌握函数概念", "理解单调性"],
  "teaching_focus": ["函数定义域", "单调性判断"],
  "teaching_difficulties": ["复合函数单调性"],
  "teaching_methods": ["讲授法", "启发式"],
  "teaching_aids": ["多媒体投影", "几何画板"],
  "teaching_process": {
    "warm_up": {"duration": "5分钟", "content": "..."},
    "introduction": {"duration": "5分钟", "content": "..."},
    "new_teaching": [{"step":1,"title":"...","duration":"...","content":"...","activities":["提问","讨论"]}],
    "practice": {"duration": "10分钟", "content": "..."},
    "summary": {"duration": "5分钟", "content": "..."},
    "homework": ["习题1", "习题2"]
  },
  "classroom_activities": [{"name":"角色扮演","type":"情景模拟","duration":"10分钟","description":"...","purpose":"..."}],
  "teaching_reflection": "本次课学生参与度较高..."
}
```

#### 实现逻辑

1. **依赖检测**：使用 `python-docx`，未安装时报错退出
2. **样式定义**：深蓝色 (`1F4E79`) 用于标题，金色 (`4472C4`) 用于子标题
3. **生成顺序**：
   - 封面页（居中标题 + 学科/年级/课时）
   - 一、教学目标（bullet list）
   - 二、教学重点与难点
   - 三、教学方法
   - 四、教学准备
   - 五、教学过程（热导入 → 新课导入 → 新授环节 → 课堂练习 → 总结 → 课后作业）
   - 六、课堂活动设计
   - 七、教学反思
4. **输出**：保存为 `.docx` 文件，stdout 返回 `{"success": true, "docx_path": "...", "error": ""}`

### 4.2 `render_game.py` — 互动小游戏渲染

#### 支持的游戏类型

| 类型 | 英文名 | 说明 |
|------|--------|------|
| 选择题 | `quiz` | 四选一，实时反馈，含解析 |
| 连连看 | `matching` | 左右词语配对，点击匹配 |
| 排序题 | `ordering` | 将打乱顺序的选项排成正确顺序 |
| 填空题 | `fill_blank` | 填词游戏，输入验证，含提示 |

#### 输入格式（stdin JSON）

```json
{
  "output_path": "/path/to/game.html",
  "game_type": "quiz",
  "title": "知识点小测验",
  "data": [
    {
      "question": "光合作用的原料是什么？",
      "options": ["氧气", "二氧化碳", "水", "氮气"],
      "answer": 1,
      "explanation": "光合作用的原料是二氧化碳和水..."
    }
  ]
}
```

#### 实现逻辑

1. **共享 CSS/JS 库**：深色背景，渐变进度条，动画过渡，响应式设计
2. **答题流程**（以 Quiz 为例）：
   - 点击选项 → 立即判定对错 → 显示正确/错误样式 → 显示解析 → 显示"下一题"按钮
   - 最后一题 → 显示结果页（圆形得分展示 + 详细答题记录）
3. **连连看**：`selectedLeft` 记录左侧选中项，点击右侧时比较索引，相等则标记 matched，不等则 shake 动画
4. **填空题**：正则替换 `[blank]` 或 `___` 为带下划线的占位符，Enter 键提交，输入正确后占位符显示答案
5. **输出**：单个独立 HTML5 文件，无需服务器，可直接在浏览器打开

### 4.3 `render_animation.py` — 动画渲染

#### 支持的动画类型

| 类型 | 效果 | 适用场景 |
|------|------|----------|
| `reveal` | 淡入 + 上移 | 文字逐条揭示 |
| `slide` | 左滑入 | 内容逐条出现 |
| `zoom` | 缩放出现 | 关键概念弹出 |
| `typewriter` | 打字机效果 | 推导过程展示 |
| `diagram` | 框图渐显 | 知识结构图 |
| `timeline` | 时间线展开 | 历史/发展脉络 |

#### 输入格式（stdin JSON）

```json
{
  "output_path": "/path/to/output.html",
  "format": "html5",
  "title": "细胞分裂动画",
  "animation_type": "reveal",
  "data": {
    "scenes": [
      {
        "id": 0,
        "elements": [
          {"type": "text", "content": "间期：DNA复制", "style": {"font_size": 24, "color": "#58a6ff"}},
          {"type": "shape", "style": {"shape": "rect", "w": 200, "h": 100, "background": "#21262d"}}
        ],
        "animation": {"type": "fade", "duration": 1.0, "delay": 0.5}
      }
    ]
  }
}
```

#### HTML5 格式实现逻辑

1. **CSS 动画定义**：每种动画类型对应一个 CSS `@keyframes`，通过 `animation-delay` 控制多元素顺序出现
2. **场景切换**：使用 `.scene-slide` div，通过 JS `showScene(idx)` 控制 `opacity` 切换
3. **进度指示器**：底部圆点导航 + 顶部进度条，支持自动播放（每 3 秒切换）和手动切换
4. **自定义 CSS**：支持传入 `css_keyframes` 字段注入自定义关键帧

#### GIF 格式实现逻辑

1. 写入临时 HTML 文件
2. 用 playwright 打开 HTML，针对每个 scene 截图
3. 每帧重复 `fps` 次以延长 GIF 中该帧的显示时长
4. 使用 PIL 将帧序列合成为 GIF，优化颜色

#### MP4 格式实现逻辑

1. 写入临时 HTML 文件
2. playwright 启动 Chromium，`page.video.start_recording()` 开始录制
3. JS 循环切换场景，每场景等待 2 秒
4. `page.video.stop_recording()` 停止，得到 `.webm` 文件
5. 若系统有 `ffmpeg`，转码为 `.mp4`（H.264）；否则保留 `.webm`

## 五、PPT Init 请求示例

### 5.1 独立调用（不联动）

```json
POST /api/v1/ppt/init
{
  "user_id": "u123",
  "topic": "光合作用",
  "subject": "生物",
  "total_pages": 8,
  "session_id": "sess_abc",
  "teaching_elements": {
    "knowledge_points": ["光合作用原理", "叶绿体结构"],
    "teaching_goals": ["理解光合作用概念"],
    "duration": "45分钟"
  }
}
```

### 5.2 联动调用（一键生成 PPT + 教案 + 动画 + 游戏）

```json
POST /api/v1/ppt/init
{
  "user_id": "u123",
  "topic": "光合作用",
  "subject": "生物",
  "description": "面向高一学生，讲解光合作用的基本原理和过程",
  "total_pages": 8,
  "session_id": "sess_abc",
  "teaching_elements": {
    "knowledge_points": ["光合作用原理", "叶绿体结构", "光反应", "暗反应"],
    "teaching_goals": ["掌握光合作用概念", "理解光反应和暗反应区别"],
    "teaching_logic": "从宏观现象引入 → 结构基础 → 原理过程 → 总结应用",
    "key_difficulties": ["光反应和暗反应的联系", "卡尔文循环"],
    "duration": "45分钟",
    "interaction_design": "小组讨论光合作用在农业生产中的应用",
    "output_formats": ["pptx", "docx", "html5"]
  },
  "auto_generate_teaching_plan": true,
  "auto_generate_content_diversity": true,
  "content_diversity_type": "both",
  "content_diversity_game_type": "quiz",
  "content_diversity_animation_style": "all"
}
```

**响应**（立即返回）：

```json
{
  "task_id": "task_xxxxx",
  "status": "processing"
}
```

**后续轮询**：

```bash
# 查询 PPT 状态
GET /api/v1/canvas/status?task_id=task_xxxxx

# 查询教案（plan_id 在 PPT 完成后可从 Task 获取）
GET /api/v1/teaching_plan/status/:plan_id

# 查询动画/游戏
GET /api/v1/content_diversity/status/:result_id
```

## 七、依赖说明

运行渲染脚本所需的 Python 依赖：

```bash
pip install python-docx           # 教案渲染
pip install playwright Pillow    # GIF/MP4 渲染（可选）
playwright install chromium       # 安装浏览器引擎
```

如未安装 `playwright`，`render_animation.py` 在 GIF/MP4 模式下会返回可读的 HTML5 文件作为降级方案。
