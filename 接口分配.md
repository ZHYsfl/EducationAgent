# Voice Agent 外部接口分配方案

基于 `API_DOCUMENTATION.md`，结合团队调整（曾晨曦和钟天贻互换任务）：

---

## 1. 曾晨曦 - PPT Agent 服务

### 外部 API（Voice Agent 调用，6个）

| 接口路径 | 方法 | 用途 |
|---------|------|------|
| `POST /api/v1/ppt/init` | POST | 初始化PPT任务（联动生成教案/动画/游戏） |
| `POST /api/v1/ppt/feedback` | POST | 发送PPT修改反馈 |
| `GET /api/v1/canvas/status` | GET | 获取画布状态 |
| `POST /api/v1/canvas/vad-event` | POST | 通知VAD事件 |
| `POST /api/v1/teaching_plan/generate` | POST | 生成配套Word教案 |
| `POST /api/v1/content_diversity/generate` | POST | 生成动画创意/互动小游戏 |

### 内部 API（PPT Agent 内部使用，外部不暴露）

| 接口路径 | 方法 | 用途 |
|---------|------|------|
| `POST /internal/feedback/generate_pages` | POST | Init 时批量并发生成页面 |
| `POST /internal/feedback/timeout_tick` | POST | 超时处理（由内部 goroutine 调用，可手动触发） |
| `POST /internal/ppt/export` | POST | 创建导出任务（.pptx / .docx / .html） |
| `GET /internal/ppt/export/:export_id` | GET | 查询导出任务状态 |
| `GET /internal/ppt/page/:page_id/render` | GET | 获取单个页面渲染结果 |
| `GET /api/v1/teaching_plan/status/:plan_id` | GET | 查询教案生成状态及下载链接 |
| `GET /api/v1/content_diversity/status/:result_id` | GET | 查询动画/游戏生成状态及内容 |
| `POST /api/v1/content_diversity/export` | POST | 导出动画/游戏（html5 / gif / mp4） |
| `POST /api/v1/content_diversity/integrate` | POST | 将动画/游戏嵌入PPT页面代码 |

> **Init 联动机制**: `POST /api/v1/ppt/init` 支持 `auto_generate_teaching_plan` 和 `auto_generate_content_diversity` 字段。设为 `true` 时，PPT 初稿生成完成后自动并发触发教案和动画/游戏的生成，无需教师单独调用。生成的 `plan_id` 和 `result_id` 会同步回 Task 记录中，可通过 `GET /api/v1/teaching_plan/status/:plan_id` 和 `GET /api/v1/content_diversity/status/:result_id` 查询。
> **冲突问答优先级说明**: `conflict_question` 消息会被 Voice Agent 强制
设为高优先级（`priority: high`），确保用户能立即看到冲突问题，不会被其他普
通消息阻塞。

## 2. 段怡鹏 - 知识库服务

- `POST /api/v1/kb/query` - 专业知识查询（voice agent 直接调用，返回 summary）
- `POST /api/v1/kb/query-chunks` - 关键词 chunk 检索（供记忆模块 / PPT Agent 调用，返回 []chunk）
- `POST /api/v1/kb/ingest-from-search` - 从搜索结果导入知识库（由搜索服务调用）

---

## 3. 王松彬 - 认证与记忆服务

**认证服务**:

- `POST /api/v1/auth/verify` - 验证用户token（WebSocket连接时）
- `GET /api/v1/auth/profile` - 获取用户基础信息

**记忆服务**（个人记忆模块为控制面）:

- `POST /api/v1/memory/recall` - 触发记忆召回（异步，立即返回 success，结果通过回调返回）
- `POST /api/v1/memory/context/push` - 接收 voice agent 推送的历史消息，上传 OSS，触发 kb 嵌入索引

> 以下接口已废弃，由记忆模块内部处理：

> - ~~`POST /api/v1/memory/extract`~~
> - ~~`POST /api/v1/memory/working/save`~~
> - ~~`GET /api/v1/memory/working/{session_id}`~~

## 4. 钟天贻 - 会话、搜索与文件服务

**会话管理**:

- `POST /api/v1/sessions` - 创建会话
- `GET /api/v1/sessions/{session_id}` - 获取会话详情
- `GET /api/v1/sessions?user_id={user_id}` - 获取用户会话列表
- `PUT /api/v1/sessions/{session_id}` - 更新会话

**搜索服务**:

- `POST /api/v1/search/query` - 网络搜索（异步，搜完后调 kb-service 的 ingest-from-search 入库）

**文件服务**（存储 OSS 元信息，文件实体在 OSS）:

- `POST /api/v1/files/upload` - 文件上传（存 OSS，返回元信息）
- `GET /api/v1/files/:file_id` - 下载/查看元数据
- `DELETE /api/v1/files/:file_id` - 删除
